# Commercial Offer Assistant

Система подготовки коммерческих предложений для дилера техники на базе `n8n`, `NocoDB` и `OpenAI`.

Проект автоматизирует обработку входящих клиентских заявок: принимает запрос через веб-форму, определяет тип обращения, подбирает подходящие позиции из каталога, формирует черновик коммерческого предложения, сохраняет историю действий и отправляет результат менеджеру на проверку.

## Что делает система

- принимает заявку клиента через HTML-форму;
- сохраняет обращение в таблицу `Requests`;
- определяет тип запроса: `техника`, `запчасти`, `сервис`, `другое`;
- выделяет ключевые требования клиента;
- подбирает подходящие позиции из таблицы `Catalog`;
- формирует HTML-черновик коммерческого предложения;
- сохраняет черновик и обновляет статус заявки;
- отправляет письмо менеджеру на проверку;
- пишет лог каждого этапа в таблицу `ActionLog`;
- защищает процесс от дублей заявок.

## Бизнес-кейс

Менеджеры дилера техники ежедневно получают разные типы запросов:

- подбор тракторов и другой техники;
- подбор запчастей;
- сервисное обслуживание и ремонт.

Обычно менеджер вручную уточняет потребность, ищет позиции, оценивает стоимость и готовит коммерческое предложение. Этот workflow сокращает ручную работу и ускоряет подготовку первого черновика для менеджера.

## Стек

- [n8n](https://n8n.io/) — orchestration и workflow-автоматизация
- [NocoDB](https://www.nocodb.com/) — хранение заявок, каталога и журнала действий
- [OpenAI](https://platform.openai.com/) — классификация запроса, AI-подбор и генерация текстовых блоков КП

## Структура данных

### `Requests`

- `request_id`
- `client_name`
- `email`
- `request_text`
- `request_type`
- `manager_comment`
- `offer_text`
- `status`

### `Catalog`

- `item_id`
- `category`
- `name`
- `description`
- `price`
- `available`

### `ActionLog`

- `request_id`
- `action`
- `datetime`
- `result`

## Как работает workflow

1. Клиент открывает страницу `/new-request`.
2. Заполняет форму и отправляет заявку.
3. Workflow сохраняет заявку в `Requests` со статусом `new`.
4. OpenAI определяет тип запроса и выделяет требования.
5. Из `Catalog` выбираются доступные позиции нужной категории.
6. AI-нодa подготавливает персонализированные текстовые блоки для КП:
   - `subject`
   - `greeting`
   - `need_summary`
   - `next_step`
   - `cta`
7. Нода `Build Offer Payload` собирает итоговое HTML-коммерческое предложение:
   - персонализированное обращение;
   - блок потребности клиента;
   - таблица только с отобранными позициями;
   - причины выбора;
   - ориентировочная стоимость;
   - следующий шаг;
   - аккуратный деловой CTA.
8. Черновик записывается обратно в `Requests`, статус меняется на `draft_ready`.
9. Менеджеру отправляется письмо с черновиком на проверку.
10. Каждый важный этап фиксируется в `ActionLog`.

## Особенности реализации

- Современная HTML-форма с удобным UX для подачи заявки.
- Антидубль-проверка для защиты от повторной отправки формы.
- Разделение ответственности:
  - AI отвечает за смысл и персонализацию текста;
  - `Build Offer Payload` отвечает за точную структуру, вывод позиций и расчёт стоимости.
- Если AI не вернул нужные текстовые блоки, workflow использует безопасные fallback-значения.
- Для нескольких альтернатив в категории `техника` итоговая стоимость выводится диапазоном `от ... до ...`, а не суммой всех вариантов.

## Как запустить

1. Создайте таблицы `Requests`, `Catalog` и `ActionLog` в `NocoDB`.
2. Импортируйте актуальный workflow в `n8n`.
3. Подключите credentials:
   - `NocoDB`
   - `OpenAI`
   - SMTP / Email node
4. Проверьте идентификаторы `workspace`, `project` и `table` в нодах `NocoDB`.
5. Убедитесь, что в `Catalog` есть данные для подбора.
6. Активируйте workflow и откройте webhook-форму.

## Ожидаемый результат

На выходе менеджер получает не сырую заявку, а готовый черновик коммерческого предложения в HTML-формате, который можно быстро проверить, при необходимости скорректировать и отправить клиенту.
